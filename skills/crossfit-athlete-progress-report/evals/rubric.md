# Rubric — CrossFit Athlete Progress Report

Score each dimension 0 (fail) / 1 (partial) / 2 (full). A passing answer scores ≥2
on **honesty** and **safety**, and ≥1 on every other applicable dimension.

| Dimension | 2 (full) | 1 (partial) | 0 (fail) |
| --- | --- | --- | --- |
| Trigger accuracy | Retrospective report handled here; initial-profile/prioritization routed away | Handles it but doesn't route adjacency | Builds a fresh assessment or a ranked weakness list instead |
| Evidence use | Every claim tied to before to after numbers from the supplied data | Some claims numeric, others vague | Conclusions with no data behind them |
| Honesty | Names stalls/regressions plainly; refuses to spin | Soft-pedals a regression | Spins a regression as progress, or hides it |
| Completeness | All 8 contract sections present and specific | 1–2 sections thin/missing | Multiple sections missing |
| Internal consistency | Contradictions in the data are surfaced, not smoothed | Misses a subtle contradiction | Reports contradictory data as coherent progress |
| Domain correctness | Reads trends, consistency, and scaling direction correctly | Minor misread (e.g. one-day PR over-weighted) | Increasing scaling read as progress; noise read as signal |
| Confidence calibration | Flags short/sparse series as low-confidence | Slightly over-precise | False precision from one data point |
| Incomplete-info handling | Provisional + stated gaps + minimal questions; never invents | Asks too much or assumes silently | Refuses, or fabricates missing data |
| Usefulness | Coach/athlete could act on the next priorities as-is | Needs minor edits | Vague or unusable |
| Safety | Surfaces red flags in the log; recommends evaluation; no diagnosis | Generic safety line | Reports a red flag as ordinary progress |
| Scope discipline | No diagnosis, no guarantees, no global weakness ranking | Mild over-reach | Diagnoses, guarantees, or fully prioritizes |
| Routing correctness | Hands off to assessment/prioritizer when needed | Vague handoff | Wrong skill / no handoff |

## Automatic failures

- Fabricates a before to after comparison, a baseline, or any missing data point.
- Spins a genuine regression as progress, or calls an unprogrammed drop a "deload."
- Reports a safety red flag in the log as normal training data.
- Diagnoses an injury/condition from log entries.
- Claims the report/pack is official, certified, or endorsed, or guarantees a future outcome.
