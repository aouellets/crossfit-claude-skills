# Example 2 — Partial profile flagging unknowns + recommended tests

**Prompt:** "What kind of athlete am I? I've been doing CrossFit about a year and
I can do most RX workouts. I want to get fitter."

**Context:** No maxes, no benchmark times, no skill list, no injuries mentioned.

---

> **Provisional profile — limited data.** You gave a training age and a general
> sense of level but no maxes, benchmark scores, or skill detail. I will rate only
> what that supports and mark the rest as **unknown**, then recommend tests to
> close the gaps. I am not inventing numbers.

## Snapshot
~1-year athlete, self-reports completing most RX workouts; capacity profile is
mostly **unmeasured**, so this is a scaffold, not a verdict.

## Profile by quality
- **Strength** — **Unknown** (no 1RMs supplied).
- **Power / weightlifting** — **Unknown**.
- **Aerobic / anaerobic capacity** — **Unknown** (no benchmark times).
- **Gymnastics / skill** — **Unknown** (no skill list); "most RX" hints at basic
  competence but is not evidence of specific skills.
- **Mobility / movement quality** — **Unknown**.
- **Work capacity / pacing / consistency / recovery** — **Unknown**.
- **Goal-fit** — Goal ("get fitter") is broad GPP, which is achievable but
  un-targeted without data.

## Strengths
None can be claimed yet on evidence — "most RX" is a useful signal but not a rating.

## Limiters
None can be named yet without data; naming one now would be a guess.

## Unknowns
Essentially every quality above. This is the main finding.

## Ranked priorities (provisional)
Cannot rank limiters that haven't been measured. First priority is to **measure**.

## Recommended tests
A short benchmark battery to convert unknowns into a real profile: one strength
lift (e.g. back squat or deadlift 3–5RM), one monostructural test (2k row or
500 m + a longer piece), one named benchmark (e.g. a girl WOD) with the scaling
used, and a gymnastics skill inventory (strict pull-ups, push-ups, double-unders,
HSPU). Hand the battery design to `crossfit-benchmark-and-testing-builder`.

## 4–12 week focus
Run the test battery in week 1, then a balanced GPP block. Reassess at week 8.

## Success metrics
Have real numbers for at least one lift, one engine test, and a gymnastics
inventory by the retest — that turns this provisional scaffold into a profile.

**To sharpen this now, the two most useful things you could share:** any one
strength number you know, and any one benchmark time.
