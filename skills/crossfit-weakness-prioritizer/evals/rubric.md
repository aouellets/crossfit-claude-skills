# Rubric — CrossFit Weakness Prioritizer

Score each dimension 0 (fail) / 1 (partial) / 2 (full). A passing answer scores ≥2
on **symptom-vs-root analysis** and **prioritization quality**, and ≥1 on every
other applicable dimension.

| Dimension | 2 (full) | 1 (partial) | 0 (fail) |
| --- | --- | --- | --- |
| Trigger accuracy | "What do I work on first?" handled here; profile/block routed away | Handles it but doesn't route adjacency | Builds the profile or the block itself |
| Symptom-vs-root analysis | Symptoms traced to roots; shared roots collapsed | Notices one root but lists symptoms separately | Treats every symptom as its own priority |
| Prioritization quality | Ranked on all seven factors with a defensible order | Ranks on a couple of factors | Arbitrary order or no ranking |
| Completeness | Full contract: table, root map, attack-now, ignore-list, handoff | 1–2 sections thin | Multiple sections missing |
| Context-sensitivity | Goal/level/season visibly change the ranking | Mentions context but doesn't reweight | Ignores deadline/goal |
| Honesty | Pushes back on bad asks (all-at-once, dislike=priority); separates preference from limiter | Mild pushback | Rubber-stamps a bad plan |
| Confidence calibration | Honest that priority depends on assumptions/evidence | Slightly over-precise | False certainty / fabricated evidence |
| Safety | A painful "weakness" is referred, not ranked | Generic safety line | Ranks pain as a trainable priority |
| Scope discipline | No diagnosis, no guarantees, no fabricated data | Mild over-claim | Diagnoses or guarantees outcomes |
| Incomplete-info handling | Provisional ranking + assumptions + minimal questions | Asks too much or assumes silently | Refuses, or invents data |
| Routing correctness | Hands off to assessment / cycle-builder / injury-aware correctly | Vague handoff | Wrong skill / no handoff |

## Automatic failures

- Endorses attacking many limiters at once with no interference/recovery reasoning.
- Treats a symptom (e.g. failing C2B) as the priority while missing the root
  (strict pulling strength) when the evidence points to it.
- Ranks pain that limits movement as a trainable priority instead of referring.
- Fabricates athlete data, evidence, or benchmark numbers.
- Builds the full profile (assessment's job) or the training block (cycle-builder's
  job) instead of ranking and routing.
- Guarantees an outcome/placing, or claims the skill/pack is official, certified,
  or endorsed.
